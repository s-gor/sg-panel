# SG-Panel v0.9.8

SG-Panel — компактная веб-панель для управления Xray Reality без сторонних панелей.

Панель хранит настройки в SQLite, формирует `config.json`, проверяет его через Xray перед применением, сохраняет резервную копию предыдущей конфигурации и только после этого перезапускает службу.

Базовая установка проверена на Amazon EC2 с Ubuntu Server 26.04 LTS и инстансом `t3.micro`. Версия `0.9.8` исправляет опасное поведение деинсталлятора: обычное удаление теперь сохраняет Xray, его конфигурацию и резервные копии.

## Возможности

- VLESS Reality inbound на TCP `443`;
- пользователи, UUID, сроки действия и учёт трафика;
- VLESS-ссылки, QR-коды и подписки;
- безопасная проверка и применение `config.json`;
- резервные копии SQLite и конфигурации Xray;
- маршрутизация, блокировки и пользовательские выходы;
- управляемый DNS Xray;
- диагностика служб, портов, ресурсов и журналов;
- защита входа, сессии администратора, IP allowlist и аудит;
- HTTPS через Nginx и Let's Encrypt;
- CLI и веб-интерфейс.

### Пользовательские выходы

В разделе **«Выходы»** можно создать несколько независимых пользовательских выходов — например `eu-exit`, `usa-exit` и `home-exit`. Каждый выход имеет собственный тег и параметры подключения к удалённому Xray-серверу.

Поддерживаются три комбинации:

- `VLESS + RAW/TCP + REALITY`;
- `VLESS + XHTTP + TLS`;
- `VLESS + XHTTP + REALITY`.

XHTTP является транспортом VLESS, а не отдельным протоколом. VMess, Trojan, Shadowsocks и другие протоколы пока не включены.

## Схема работы

```text
ДОМЕН:443       Xray Reality
ДОМЕН:61443     Nginx HTTPS → SG-Panel 127.0.0.1:8080
ДОМЕН:80        HTTP-заглушка и Let's Encrypt HTTP-01
```

Один домен можно использовать одновременно для Xray и панели: службы работают на разных портах.

Порт `8080` является внутренним. Открывать его в AWS Security Group или firewall нельзя.

## Требования

- чистый сервер Ubuntu;
- доступ `sudo` или `root`;
- публичный IPv4;
- доменное имя с A-записью на публичный IPv4 сервера;
- открытые входящие порты.

Dynu использовать необязательно. Подходит любой DNS-провайдер, позволяющий создать A-запись.

Установка непосредственно на IP без домена в версии `0.9.8` не поддерживается.

## Порты

| Порт | Назначение | Рекомендуемый источник |
|---:|---|---|
| `22/tcp` | SSH | ваш публичный IP `/32` |
| `80/tcp` | HTTP-заглушка и Let's Encrypt | `0.0.0.0/0` |
| `443/tcp` | Xray Reality | `0.0.0.0/0` |
| `61443/tcp` | SG-Panel HTTPS | ваш публичный IP `/32` |
| `8080/tcp` | внутренний backend | не открывать |

Порт панели можно изменить во время установки. Допустимый диапазон: `49152–65535`.

## Подготовка DNS

До запуска установщика A-запись домена должна указывать на текущий публичный IPv4 сервера.

Проверка с компьютера:

```powershell
nslookup vpn.example.dynu.net
```

Проверка на сервере:

```bash
curl -4 -s https://checkip.amazonaws.com
getent ahostsv4 vpn.example.dynu.net
```

Публичный IPv4 сервера и адрес из DNS должны совпадать.

При использовании обычного публичного адреса EC2 он может измениться после остановки и повторного запуска инстанса. В этом случае сначала обновите A-запись.

## Установка из GitHub

Скопируйте команду целиком, вставьте в терминал и нажмите Enter:

```bash
sudo apt-get update && sudo apt-get install -y curl ca-certificates unzip && curl -fsSL https://raw.githubusercontent.com/s-gor/sg-panel/main/install-from-github.sh -o /tmp/install-sg-panel.sh && bash -n /tmp/install-sg-panel.sh && chmod 700 /tmp/install-sg-panel.sh && sudo bash /tmp/install-sg-panel.sh
```

Не запускайте интерактивный мастер через `curl | bash`: установщику требуется обычный ввод с клавиатуры.

### Установка из ZIP-релиза

После загрузки и распаковки полного архива:

```bash
cd SG-Panel-v0.9.8
sudo bash deploy/ec2-first-install.sh
```

Этот способ использует тот же мастер чистой установки и тот же автоматический режим обновления.

Установщик запросит:

```text
Домен Xray-сервера
Домен HTTPS-панели
Внешний HTTPS-порт панели [61443]
Имя первого пользователя [sg-admin]
Reality target [www.microsoft.com:443]
Reality SNI [www.microsoft.com]
Пароль администратора панели
```

Значение в квадратных скобках является рекомендуемым. Чтобы принять его, просто нажмите Enter.

Пароль администратора должен содержать не менее восьми символов.

Для Xray и панели можно использовать один и тот же домен.

### Что делает установщик

Установщик:

1. проверяет исходники SG-Panel;
2. при малом объёме RAM создаёт swap `2 ГиБ`;
3. устанавливает Xray `26.3.27`, SG-Panel, Nginx и Certbot;
4. проверяет DNS;
5. создаёт Reality-ключи и первого пользователя;
6. формирует и проверяет конфигурацию Xray;
7. запускает backend только на `127.0.0.1:8080`;
8. получает или повторно использует сертификат Let's Encrypt;
9. настраивает HTTPS панели и проверяет её готовность;
10. сохраняет VLESS-ссылку первого пользователя в `/root/sg-panel-first-user.txt`.

При ошибке Certbot установщик показывает понятную причину: неправильный DNS, закрытый порт `80`, ошибка подтверждения домена или лимит Let's Encrypt.

Удаление старого сертификата не сбрасывает лимит Let's Encrypt.

## После установки

Панель открывается по адресу:

```text
https://ВАШ-ДОМЕН:61443
```

VLESS-ссылка первого пользователя:

```bash
sudo cat /root/sg-panel-first-user.txt
```

Проверка служб:

```bash
systemctl is-active xpanel-web
systemctl is-active xray
systemctl is-active nginx
systemctl is-active xpanel-maintenance.timer
```

Проверка версии:

```bash
cd /opt/xpanel-mvp
.venv/bin/python -m xpanel --version
```

Ожидаемый результат:

```text
xpanel 0.9.8
```

## Первый вход

В SG-Panel нет отдельного имени администратора: вход выполняется по паролю, заданному во время установки.

Первый пользователь `sg-admin` является обычным пользователем Xray и не является учётной записью администратора панели.

## Защита входа и снятие блокировки

По умолчанию:

- после пяти неверных попыток блокируется только IP-адрес источника;
- блокировка действует 15 минут;
- другие IP продолжают получать доступ;
- после успешного входа счётчик сбрасывается;
- даже правильный пароль не позволит войти, пока блокировка действует.

Снять блокировку со всех IP:

```bash
cd /opt/xpanel-mvp && sudo .venv/bin/python -m xpanel unlock-admin
```

Снять блокировку только с одного IP:

```bash
cd /opt/xpanel-mvp && sudo .venv/bin/python -m xpanel unlock-admin --ip 203.0.113.10
```

Во время проверки защиты не закрывайте единственную SSH-сессию.

## Использование панели

После входа проверьте главную страницу: службы SG-Panel, Nginx и Xray должны иметь состояние `active`.

Для каждого человека или устройства рекомендуется создавать отдельного пользователя.

После добавления, изменения, включения или отключения пользователя нажмите **«Применить»**, чтобы обновить рабочую конфигурацию Xray.

Для обычного сервера без пользовательских выходов оставьте:

```text
Default outbound: direct
Domain strategy:  AsIs
Sniffing:         включён
Route only:       включён
HTTP:             включён
TLS:              включён
QUIC:             включён
```

## Страница-заглушка

SG-Panel устанавливает простую статическую страницу-заглушку в цветах панели.

В текущей стабильной конфигурации она доступна только по HTTP:

```text
http://ваш-домен/
```

Адрес:

```text
https://ваш-домен/
```

не используется для сайта, потому что порт `443` полностью занят Xray Reality. Это не ошибка установки.

Панель доступна по адресу:

```text
https://ваш-домен:61443/
```

Рабочий файл страницы:

```text
/var/www/sg-panel-placeholder/index.html
```

Эталонная версия:

```text
/var/www/sg-panel-placeholder/index.default.html
```

Восстановить стандартный вариант:

```bash
sudo cp /var/www/sg-panel-placeholder/index.default.html /var/www/sg-panel-placeholder/index.html
sudo nginx -t && sudo systemctl reload nginx
```

Одновременная работа HTTPS-сайта и Xray Reality на публичном порту `443` требует дополнительного SNI-разделения через Nginx Stream или HAProxy.

Этот режим будет рассмотрен после подтверждения стабильной работы основной архитектуры SG-Panel при установке, обновлении, удалении, восстановлении и обычной эксплуатации.

HTTPS-заглушка на порту `443` пока не входит в стандартную конфигурацию SG-Panel.

## v2rayNG и TUN

Импортируйте VLESS-ссылку или QR-код из раздела **«Пользователи»**.

Для работы TUN в v2rayNG:

1. выберите режим VPN;
2. включите `Enable Hev TUN Feature`;
3. на время проверки отключите per-app proxy;
4. используйте глобальную маршрутизацию.

Серверный DNS SG-Panel для обычного подключения включать не требуется. TUN DNS обрабатывается клиентом.

## Оптимизация для t3.micro

Для сервера с `1 ГиБ` RAM установщик:

- создаёт swap `2 ГиБ`;
- запускает Waitress с двумя потоками;
- выполняет обслуживание раз в 15 минут;
- оставляет статистику Xray выключенной по умолчанию;
- загружает библиотеку QR только при открытии QR-кода.

Для личного сервера и небольшого количества пользователей `t3.micro` достаточно. При высокой одновременной нагрузке используйте более мощный инстанс.

## Обновление

Запустите ту же команду установки из GitHub. Установщик сам обнаружит существующую SG-Panel и перейдёт в режим обновления без повторных вопросов.

При обновлении не меняются домен, HTTPS-порт, сертификат, пароль администратора и настройки Xray. Сохраняются база, пользователи, Reality-ключи, маршрутизация, DNS, подписки и настройки безопасности.

Перед заменой файлов создаётся резервная копия. При ошибке выполняется rollback.

## Удаление

Обычное удаление в версии `0.9.8` безопасно по умолчанию: оно удаляет только SG-Panel, её службы и созданную панелью конфигурацию Nginx.

Сохраняются:

- Xray, его служба и текущий `config.json`;
- резервные копии `/root/sg-panel-backups`;
- Nginx, Certbot и сертификаты Let's Encrypt;
- системные пакеты и `/swapfile`.

Перед удалением создаётся финальная копия базы и конфигурации Xray.

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-panel/main/deploy/uninstall.sh -o /tmp/uninstall-sg-panel.sh && bash -n /tmp/uninstall-sg-panel.sh && chmod 700 /tmp/uninstall-sg-panel.sh && sudo bash /tmp/uninstall-sg-panel.sh
```

Удаление Xray или резервных копий возможно только через явные параметры:

```bash
sudo bash /tmp/uninstall-sg-panel.sh --remove-xray
sudo bash /tmp/uninstall-sg-panel.sh --remove-backups
sudo bash /tmp/uninstall-sg-panel.sh --remove-xray --remove-backups
```

Старый параметр `--purge-all` удалён из обычного деинсталлятора. Полная очистка одноразового тестового сервера вынесена в отдельный скрипт и не должна применяться на сервере с другими сайтами или сертификатами.

Подробное описание всех режимов:

[Безопасное удаление SG-Panel](docs/UNINSTALL.md)

## Диагностика

Статус служб:

```bash
systemctl --no-pager --full status xpanel-web xray nginx
```

Журнал панели:

```bash
journalctl -u xpanel-web -n 100 --no-pager
```

Журнал Xray:

```bash
journalctl -u xray -n 100 --no-pager
```

Ошибки Nginx:

```bash
tail -n 100 /var/log/nginx/error.log
```

Проверка backend:

```bash
curl -sS -o /dev/null -w 'Backend HTTP: %{http_code}\n' http://127.0.0.1:8080/login
```

Проверка HTTPS из самого сервера:

```bash
curl -k -sS -o /dev/null -w 'HTTPS: %{http_code}\n' https://127.0.0.1:61443/login -H 'Host: ВАШ-ДОМЕН'
```

Код `502` означает, что Nginx работает, но backend `127.0.0.1:8080` ещё не готов или не запущен.

## Сертификат Let's Encrypt

Проверка автоматического продления без выпуска нового сертификата:

```bash
sudo certbot renew --dry-run
```

Для HTTP-01 порт `80/tcp` должен быть доступен из интернета.

## Безопасность

- не открывайте `8080/tcp` во внешней сети;
- ограничьте SSH и порт панели своим публичным IP `/32`;
- используйте уникальный пароль администратора длиной не менее восьми символов;
- не публикуйте `/root/sg-panel-first-user.txt`, приватный Reality-ключ и резервные копии;
- применяйте конфигурацию только через SG-Panel;
- перед крупными изменениями создавайте резервную копию.

## Основные пути

```text
/opt/xpanel-mvp                    приложение и виртуальное окружение
/opt/xpanel-mvp/data/panel.db      база SQLite
/usr/local/etc/xray/config.json    активная конфигурация Xray
/etc/xpanel-mvp/web.env            настройки веб-службы
/root/sg-panel-backups             резервные копии обновлений
/root/sg-panel-first-user.txt      первая VLESS-ссылка
```

## Текущий статус

SG-Panel `0.9.4` прошла проверку чистой установки на EC2, входа в GUI, выпуска сертификата, работы Xray Reality и подключения v2rayNG в обычном режиме и через Hev TUN.

Версия `0.9.7` добавила единый интерфейс и пользовательские VLESS-выходы. Версия `0.9.8` делает удаление безопасным по умолчанию и выносит полную очистку тестового сервера в отдельный скрипт.

Перед публикацией `0.9.8` нужно проверить обычное удаление с сохранением Xray, повторную установку и обновление существующей `0.9.7` через GitHub-команду.
