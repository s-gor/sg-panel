# SG-Panel v0.10.0 RC9

SG-Panel — компактная self-hosted веб-панель для управления Xray на собственном сервере.

Панель хранит настройки в SQLite, формирует `config.json`, проверяет его через `xray run -test`, создаёт резервную копию и только после этого перезапускает Xray. Для обычных операций используется одна кнопка **«Сохранить и применить»**.

RC9 не меняет проверенную сетевую схему RC8. Основные изменения относятся к документации, логике навигации и устранению последнего несогласованного действия при создании Outbound из JSON.

## Проверенные возможности

На чистом Amazon EC2 подтверждены:

- установка SG-Panel из GitHub;
- `RAW/TCP + REALITY`;
- `XHTTP + TLS`;
- `XHTTP + REALITY`;
- Cloudflare WARP для всего трафика;
- создание и отключение пользователей;
- прямые VLESS-ссылки и QR-коды;
- персональные HTTPS-подписки для v2rayN;
- переключение Inbound без изменения UUID пользователей;
- автоматическая проверка, резервная копия и откат при ошибке применения.

## Основные разделы панели

| Раздел | Назначение |
|---|---|
| **Обзор** | Краткое состояние Xray, текущего Inbound, Default Outbound и пользователей. |
| **Пользователи** | Создание пользователей, UUID, сроки действия, прямые ссылки и QR-коды. |
| **Подписки** | Персональные HTTPS-URL подписок и управление токенами. |
| **Inbound** | Как клиент подключается к серверу. |
| **Outbounds** | Какие выходы доступны Xray: `direct`, `blocked`, `warp`, пользовательские серверы. |
| **Routing** | Какой трафик направляется в каждый Outbound. |
| **DNS** | Внутренний DNS Xray и специальные DNS-правила. |
| **Xray Config** | Предпросмотр и расширенное редактирование итогового JSON. |
| **Diagnostics** | Службы, WARP, порты, журналы и диагностический отчёт. |
| **Резервные копии** | Ручные копии SQLite и текущего `config.json`. |
| **Безопасность** | Пароль, сессии, allowlist и журнал действий. |

Начните со страницы [С чего начать](docs/START-HERE.md). Там описана логика панели и обычные рабочие сценарии.

## Сетевые режимы

SG-Panel поддерживает три основных Inbound-профиля:

| Профиль | Публичный порт `443` | Сертификат Xray | Flow |
|---|---|---|---|
| `RAW/TCP + REALITY` | Xray | не используется | `xtls-rprx-vision` |
| `XHTTP + TLS` | Nginx, затем Xray `127.0.0.1:8443` | Let's Encrypt | отсутствует |
| `XHTTP + REALITY` | Xray | не используется | отсутствует |

Подробные поля и порядок переключения: [Основные Inbound-профили](docs/INBOUND-PROFILES.md).

Полные пути трафика для `direct`, WARP и пользовательского Outbound: [Схемы движения трафика](docs/TRAFFIC-FLOWS.md).

## Требования

- чистый сервер Ubuntu;
- `sudo` или `root`;
- публичный IPv4;
- домен с A-записью на публичный IPv4 сервера;
- открытые входящие порты `80/tcp`, `443/tcp` и HTTPS-порт панели;
- исходящий UDP для Cloudflare WARP.

Установка только по IP без домена не поддерживается.

## Порты

| Порт | Назначение | Доступ извне |
|---:|---|---|
| `22/tcp` | SSH | только ваш IP `/32` |
| `80/tcp` | HTTP-заглушка и Let's Encrypt HTTP-01 | `0.0.0.0/0` |
| `443/tcp` | основной VLESS Inbound | `0.0.0.0/0` |
| `61443/tcp` | HTTPS-панель | только ваш IP `/32` |
| `8080/tcp` | внутренний backend SG-Panel | не открывать |
| `8443/tcp` | локальный Xray для `XHTTP + TLS` | не открывать |

Порт панели выбирается при установке. Допустимый диапазон: `49152–65535`.

## Быстрая установка из GitHub

Сначала подготовьте DNS и Security Group. Полная инструкция: [Чистая установка](docs/INSTALLATION.md).

Команда установки:

```bash
sudo apt-get update && sudo apt-get install -y curl ca-certificates unzip && curl -fsSL https://raw.githubusercontent.com/s-gor/sg-panel/main/install-from-github.sh -o /tmp/install-sg-panel.sh && bash -n /tmp/install-sg-panel.sh && chmod 700 /tmp/install-sg-panel.sh && sudo bash /tmp/install-sg-panel.sh
```

Не используйте `curl | bash`: мастер установки должен читать ответы с клавиатуры.

Установщик запросит:

```text
Домен Xray-сервера
Домен HTTPS-панели
Внешний HTTPS-порт панели [61443]
Имя первого пользователя [sg-admin]
Reality target [www.bing.com:443]
Reality SNI [www.bing.com]
Пароль администратора панели
```

После завершения панель доступна по адресу:

```text
https://ВАШ-ДОМЕН:61443
```

Ссылка первого пользователя:

```bash
sudo cat /root/sg-panel-first-user.txt
```

Проверка:

```bash
systemctl is-active xpanel-web
systemctl is-active xray
systemctl is-active nginx
cd /opt/xpanel-mvp && .venv/bin/python -m xpanel --version
```

Ожидается:

```text
active
active
active
xpanel 0.10.0-rc9
```

## Обычная работа

Для большинства действий отдельная кнопка применения не нужна:

1. откройте нужный раздел;
2. измените параметры;
3. нажмите основную кнопку страницы;
4. дождитесь сообщения: `Настройки сохранены и применены. Xray работает.`

Создание, изменение, включение и удаление пользователей, Outbounds, Routing и DNS применяются автоматически.

Исключения:

- **Подписки** и **Безопасность** не меняют `config.json`, поэтому перезапуск Xray не требуется;
- **Проверить WARP** выполняет отдельный тест и не меняет Routing;
- полный редактор **Xray Config** намеренно работает в два этапа: сначала синхронизация JSON с панелью, затем отдельная проверка и применение;
- восстановление базы из резервной копии требует проверки настроек перед последующим применением.

## Обновление

Повторно запустите команду установки из GitHub. Установщик обнаружит существующую SG-Panel и выполнит обновление без повторного мастера. Если обнаружено незавершённое состояние установки, мастер откроется снова и позволит исправить домен или HTTPS-настройки.

Сохраняются:

- база SQLite;
- пользователи и UUID;
- выбранный Inbound;
- Reality-ключи;
- WARP;
- Outbounds и Routing;
- DNS;
- подписки;
- настройки безопасности;
- домен и HTTPS-порт панели.

Перед обновлением создаётся резервная копия. При ошибке выполняется rollback.

Для сознательной смены домена, HTTPS-порта панели или базовых Reality-параметров запустите сохранённый загрузчик с параметром:

```bash
sudo bash /tmp/install-sg-panel.sh --reconfigure
```

## Удаление

Обычное удаление безопасно по умолчанию: оно удаляет SG-Panel и её службы, но сохраняет Xray, Nginx, сертификаты и резервные копии:

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-panel/main/deploy/uninstall.sh -o /tmp/uninstall-sg-panel.sh && bash -n /tmp/uninstall-sg-panel.sh && chmod 700 /tmp/uninstall-sg-panel.sh && sudo bash /tmp/uninstall-sg-panel.sh
```

Для полной очистки одноразового тестового EC2 используется отдельный скрипт:

```bash
sudo bash deploy/purge-test-server.sh --destroy-test-server
```

Не запускайте полную очистку на сервере с другими сайтами или сертификатами.

Подробно: [Безопасное удаление](docs/UNINSTALL.md).

## Безопасность

- не открывайте `8080/tcp` и `8443/tcp` во внешней сети;
- ограничьте SSH и порт панели своим публичным IP `/32`;
- не публикуйте приватный Reality-ключ, WARP-ключи, токены подписок и резервные копии;
- для каждого человека или устройства создавайте отдельного пользователя;
- перед крупными изменениями создавайте резервную копию.
