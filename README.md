# SG-Panel v0.9.4

SG-Panel — компактная веб-панель для управления Xray Reality без сторонних панелей.

Панель хранит настройки в SQLite, формирует `config.json`, проверяет его через Xray перед применением, сохраняет резервную копию предыдущей конфигурации и только после этого перезапускает службу.

Версия `0.9.4` проверена на Amazon EC2 с Ubuntu Server 26.04 LTS и инстансом `t3.micro`.

## Возможности

- VLESS Reality inbound на TCP `443`;
- создание, отключение и удаление пользователей;
- UUID, сроки действия и учёт трафика;
- клиентские VLESS-ссылки и QR-коды;
- постоянные ссылки подписок;
- предварительная проверка и безопасное применение `config.json`;
- резервные копии SQLite и конфигурации Xray;
- маршрутизация и VLESS Reality outbounds;
- каскад через другой Xray-сервер;
- управляемый DNS Xray;
- диагностика служб, портов, ресурсов и журналов;
- защита входа, сессии администратора, IP allowlist и аудит;
- HTTPS через Nginx и Let's Encrypt;
- CLI и веб-интерфейс.

## Схема работы

```text
ДОМЕН:443
    Xray Reality

ДОМЕН:61443
    Nginx HTTPS
    SG-Panel 127.0.0.1:8080

ДОМЕН:80
    Статическая страница-заглушка
    Let's Encrypt HTTP-01
```

Один домен можно использовать одновременно для Xray и панели: службы работают на разных портах.

Порт `8080` является внутренним. Открывать его в AWS Security Group или firewall нельзя.

## Требования

- чистый сервер Ubuntu;
- доступ `sudo` или `root`;
- публичный IPv4;
- доменное имя с A-записью на публичный IPv4 сервера;
- открытые входящие порты.

Dynu использовать необязательно. Подходит любой DNS-провайдер, позволяющий создать A-запись.

Установка непосредственно на IP без домена в версии `0.9.4` не поддерживается. Домен нужен для HTTPS-сертификата панели.

## Порты

| Порт | Назначение | Рекомендуемый источник |
|---:|---|---|
| `22/tcp` | SSH | ваш публичный IP `/32` |
| `80/tcp` | Let's Encrypt HTTP-01 | `0.0.0.0/0` |
| `443/tcp` | Xray Reality | `0.0.0.0/0` |
| `61443/tcp` | SG-Panel HTTPS | ваш публичный IP `/32` |
| `8080/tcp` | внутренний backend | не открывать |

Порт панели можно изменить во время установки. Допустимый диапазон: `49152–65535`.

Выбранный порт автоматически добавляется в `net.ipv4.ip_local_reserved_ports`, чтобы Linux не использовал его как временный исходящий порт.

## Страница-заглушка

SG-Panel устанавливает простую статическую страницу-заглушку в цветах панели.

В текущей стабильной конфигурации она доступна только по HTTP:

```text
http://ваш-домен/
```

Это связано с распределением портов:

| Порт    | Назначение                                                  |
| ------- | ----------------------------------------------------------- |
| `80`    | страница-заглушка и проверка Let’s Encrypt                  |
| `443`   | Xray Reality                                                |
| `61443` | HTTPS-интерфейс SG-Panel                                    |
| `8080`  | внутренний backend панели, доступный только через localhost |

Поэтому адрес:

```text
https://ваш-домен/
```

в текущей версии не используется для отображения сайта. Порт `443` полностью занят Xray Reality. Это не ошибка установки.

Панель управления доступна по адресу:

```text
https://ваш-домен:61443/
```

Стандартную заглушку можно оставить без изменений или заменить своей статической страницей:

```text
/var/www/sg-panel-placeholder/index.html
```

После замены файла проверьте конфигурацию и перезагрузите Nginx:

```bash
sudo nginx -t && sudo systemctl reload nginx
```

Возможность одновременно размещать HTTPS-сайт и Xray Reality на одном публичном порту `443` требует дополнительного разделения TLS-трафика по SNI через Nginx Stream или HAProxy.

Этот режим будет рассмотрен позднее, после завершения тестирования основной архитектуры SG-Panel и подтверждения её стабильной работы при установке, обновлении, удалении, восстановлении и обычной эксплуатации.

На текущем этапе приоритетом являются:

* надёжная установка на чистый сервер;
* корректное обновление без потери данных;
* стабильная работа Xray Reality;
* безопасный доступ к панели;
* управление пользователями и конфигурацией;
* понятная диагностика ошибок;
* предсказуемое удаление и восстановление.

HTTPS-заглушка на порту `443` пока не входит в стандартную конфигурацию SG-Panel.

## Подготовка DNS

До запуска установщика A-запись домена должна указывать на текущий публичный IPv4 сервера.

Проверка с компьютера:

```powershell
nslookup vpn.example.dynu.net
```

Проверка на сервере:

```bash
curl -4 -s https://checkip.amazonaws.com
getent ahostsv4 vpn.example.dynu.net
```

Публичный IPv4 EC2 и адрес из DNS должны совпадать.

При использовании обычного публичного адреса EC2 он может измениться после остановки и повторного запуска инстанса. В этом случае обновите A-запись перед запуском SG-Panel или выпуском сертификата.

## Установка из GitHub

Скопируйте команду целиком, вставьте в терминал и нажмите Enter:

```bash
sudo apt-get update && sudo apt-get install -y curl ca-certificates unzip && curl -fsSL https://raw.githubusercontent.com/s-gor/sg-panel/main/install-from-github.sh -o /tmp/install-sg-panel.sh && bash -n /tmp/install-sg-panel.sh && chmod 700 /tmp/install-sg-panel.sh && sudo bash /tmp/install-sg-panel.sh
```

Не запускайте интерактивный мастер через `curl | bash`: установщику требуется обычный ввод с клавиатуры.

Установщик запросит:

```text
Домен Xray-сервера
Домен HTTPS-панели
Внешний HTTPS-порт панели [61443]
Имя первого пользователя [Sergey]
Reality target [www.microsoft.com:443]
Reality SNI [www.microsoft.com]
Пароль администратора панели
```

Для Xray и панели можно указать один и тот же домен.

## Что делает установщик

1. Проверяет исходники SG-Panel `0.9.4`.
2. На сервере с малым объёмом RAM создаёт swap `2 ГиБ`.
3. Устанавливает системные зависимости.
4. Определяет публичный IPv4.
5. Проверяет A-записи обоих доменов.
6. Устанавливает Xray `26.3.27`.
7. Устанавливает SG-Panel в `/opt/xpanel-mvp`.
8. Запускает backend только на `127.0.0.1:8080`.
9. Создаёт Reality-ключи и первого пользователя.
10. Формирует и проверяет конфигурацию Xray.
11. Запускает Xray Reality на `443`.
12. Устанавливает статическую страницу-заглушку и настраивает Nginx на `80` для HTTP-01.
13. Получает сертификат Let's Encrypt без запроса email.
14. Настраивает HTTPS панели на выбранном порту.
15. Ожидает готовность backend и HTTPS до 30 секунд.
16. Проверяет версии, службы и страницу входа.
17. Сохраняет VLESS-ссылку первого пользователя в `/root/sg-panel-first-user.txt`.

Если действующий сертификат для домена уже существует, установщик использует его повторно.

При ошибке Certbot установщик выводит понятную причину: неправильный DNS, закрытый порт `80`, ошибка подтверждения домена или лимит Let's Encrypt.

Удаление старого сертификата не сбрасывает лимиты Let's Encrypt.

## После установки

Панель открывается по адресу:

```text
https://ВАШ-ДОМЕН:61443
```

VLESS-ссылка первого пользователя:

```bash
sudo cat /root/sg-panel-first-user.txt
```

Проверка служб:

```bash
systemctl is-active xpanel-web
systemctl is-active xray
systemctl is-active nginx
systemctl is-active xpanel-maintenance.timer
```

Проверка версии:

```bash
cd /opt/xpanel-mvp
.venv/bin/python -m xpanel --version
```

Ожидаемый результат:

```text
xpanel 0.9.4
```

Проверка портов:

```bash
ss -lntp | grep -E ':(80|443|8080|61443)[[:space:]]'
```

Ожидаемая схема:

```text
0.0.0.0:80       nginx
0.0.0.0:443      xray
127.0.0.1:8080   waitress-serve
0.0.0.0:61443    nginx
```

## Страница-заглушка

В комплект SG-Panel входит простая статическая страница в тех же тёмно-красных цветах, что и панель управления. Она не содержит упоминаний Xray, VPN или панели и открывается по обычному HTTP:

```text
http://ВАШ-ДОМЕН
```

Порт `443` остаётся у Xray Reality, а SG-Panel работает на отдельном HTTPS-порту, по умолчанию `61443`. Поэтому встроенная заглушка обслуживается Nginx на порту `80`; каталог HTTP-01 для Let’s Encrypt продолжает работать отдельно.

Рабочий файл страницы:

```text
/var/www/sg-panel-placeholder/index.html
```

Эталонная версия из установленного пакета:

```text
/var/www/sg-panel-placeholder/index.default.html
```

Вы можете оставить стандартную страницу, заменить её собственным статическим сайтом или отредактировать `index.html`. Обновление SG-Panel не перезаписывает пользовательский `index.html`, но обновляет `index.default.html`.

Восстановить стандартный вариант:

```bash
sudo cp /var/www/sg-panel-placeholder/index.default.html /var/www/sg-panel-placeholder/index.html
sudo nginx -t && sudo systemctl reload nginx
```

После изменения страницы:

```bash
sudo nginx -t && sudo systemctl reload nginx
```

## v2rayNG и TUN

Импортируйте VLESS-ссылку или QR-код из раздела «Пользователи».

Для работы TUN в v2rayNG:

1. выберите режим VPN;
2. включите `Enable Hev TUN Feature`;
3. на время проверки отключите per-app proxy;
4. используйте глобальную маршрутизацию.

Серверный раздел DNS в SG-Panel для обычного подключения включать не требуется. TUN DNS обрабатывается клиентом.

## Оптимизация для t3.micro

Установщик автоматически учитывает сервер с `1 ГиБ` RAM:

- создаёт swap `2 ГиБ`;
- запускает Waitress с двумя потоками;
- выполняет обслуживание раз в 15 минут;
- оставляет статистику Xray выключенной по умолчанию;
- загружает библиотеку QR только при открытии QR-кода.

Для личного сервера и небольшого количества пользователей `t3.micro` достаточно. При высокой одновременной нагрузке используйте `t3.small` или более мощный инстанс.

## Обновление

Запускайте установку из GitHub той же командой. Скрипт обновления сохраняет:

- SQLite-базу;
- пользователей;
- Reality-ключи;
- маршрутизацию и outbounds;
- настройки DNS;
- подписки;
- настройки безопасности;
- пароль администратора;
- текущую конфигурацию Xray.

Перед заменой файлов создаётся резервная копия. При ошибке выполняется rollback.

## Обычное удаление

Удаляет SG-Panel, её службы, созданную панелью конфигурацию Nginx и Xray.

Nginx, Certbot, сертификаты Let's Encrypt, системные пакеты и `/swapfile` сохраняются.

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-panel/main/deploy/uninstall.sh -o /tmp/uninstall-sg-panel.sh && bash -n /tmp/uninstall-sg-panel.sh && chmod 700 /tmp/uninstall-sg-panel.sh && sudo bash /tmp/uninstall-sg-panel.sh --yes
```

## Полное удаление

Полностью очищает тестовый EC2 перед новой установкой:

- SG-Panel и базу данных;
- Xray и его конфигурацию;
- Nginx и Certbot;
- все сертификаты Let's Encrypt на этом EC2;
- ACME-каталоги;
- установленную страницу-заглушку;
- swap, созданный установщиком;
- резервирование порта панели;
- резервные и временные файлы SG-Panel.

SSH, системная сеть и правила AWS Security Group сохраняются.

```bash
curl -fsSL https://raw.githubusercontent.com/s-gor/sg-panel/main/deploy/uninstall.sh -o /tmp/uninstall-sg-panel.sh && bash -n /tmp/uninstall-sg-panel.sh && chmod 700 /tmp/uninstall-sg-panel.sh && sudo bash /tmp/uninstall-sg-panel.sh --purge-all --yes
```

Внимание: режим `--purge-all` удаляет каталог `/etc/letsencrypt` целиком. Не используйте его на сервере, где Nginx или Let's Encrypt обслуживают другие сайты.

## Диагностика

Статус служб:

```bash
systemctl --no-pager --full status xpanel-web xray nginx
```

Журнал панели:

```bash
journalctl -u xpanel-web -n 100 --no-pager
```

Журнал Xray:

```bash
journalctl -u xray -n 100 --no-pager
```

Ошибки Nginx:

```bash
tail -n 100 /var/log/nginx/error.log
```

Проверка backend:

```bash
curl -sS -o /dev/null -w 'Backend HTTP: %{http_code}\n' http://127.0.0.1:8080/login
```

Проверка HTTPS из самого сервера:

```bash
curl -k -sS -o /dev/null -w 'HTTPS: %{http_code}\n' https://127.0.0.1:61443/login -H 'Host: ВАШ-ДОМЕН'
```

Код `502` означает, что Nginx работает, но backend `127.0.0.1:8080` ещё не готов или не запущен.

## Сертификат Let's Encrypt

Проверка автоматического продления без выпуска нового сертификата:

```bash
sudo certbot renew --dry-run
```

Для HTTP-01 порт `80/tcp` должен быть доступен из интернета.

При ошибке выпуска сначала проверьте:

```bash
curl -4 -s https://checkip.amazonaws.com
getent ahostsv4 ВАШ-ДОМЕН
ss -ltnp | grep ':80 '
```

## Безопасность

- Не открывайте `8080/tcp` во внешней сети.
- Ограничьте SSH и порт панели своим публичным IP `/32`.
- Используйте уникальный пароль администратора длиной не менее восьми символов.
- Не публикуйте `/root/sg-panel-first-user.txt`, приватный Reality-ключ и резервные копии.
- После изменения настроек применяйте конфигурацию через кнопку `Apply config`.
- Перед перезапуском SG-Panel проверяет новую конфигурацию Xray и сохраняет предыдущую версию.
