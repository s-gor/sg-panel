# Резервные копии и восстановление

## Что сохраняется

Ручная резервная копия содержит:

```text
panel.db
текущий config.json
```

В `panel.db` находятся чувствительные данные:

- пользователи и UUID;
- Reality private key;
- WARP-учётные данные;
- токены подписок;
- настройки безопасности;
- Inbound, Outbounds, Routing и DNS.

Не публикуйте резервные копии и не отправляйте их без необходимости.

## Создание копии

Откройте:

```text
Резервные копии — Создать копию
```

Создание копии не меняет Xray и не требует применения конфигурации.

Рекомендуется создавать копию перед:

- редактированием полного `config.json`;
- большими изменениями Routing;
- экспериментами с DNS;
- удалением нескольких пользователей;
- изменением нескольких Outbounds.

## Скачивание

Для каждой точки доступны:

```text
Скачать DB
Скачать JSON
```

Храните файлы в зашифрованном или защищённом каталоге.

## Восстановление

Кнопка **«Восстановить DB»** заменяет текущую SQLite-базу выбранной копией.

Перед заменой SG-Panel создаёт страховочную копию текущей базы:

```text
/root/sg-panel-backups/pre-restore-ДАТА.db
```

Важное ограничение:

```text
восстановление базы не заменяет работающий config.json автоматически
```

После восстановления:

1. откройте **Inbound** и проверьте профиль;
2. откройте **Outbounds** и проверьте выходы;
3. откройте **Routing** и проверьте Default Outbound и правила;
4. откройте **Xray Config**;
5. выполните проверку;
6. примените конфигурацию.

Такой отдельный шаг оставлен намеренно: восстановленная база может относиться к другой сетевой схеме или старому домену.

## Что делать, если восстановлена неправильная копия

Найдите созданный файл `pre-restore-...db` в каталоге резервных копий и восстановите его через интерфейс либо вручную после остановки панели.

Для ручной операции сначала сохраните текущую базу и остановите веб-службу.

## Удаление копии

Кнопка **«Удалить»** удаляет файлы выбранной точки восстановления.

Удаление точки не меняет текущую базу и рабочий Xray.

## Автоматические копии

SG-Panel также создаёт копии перед:

- применением конфигурации;
- обновлением панели;
- обычным удалением;
- восстановлением базы.

Основной каталог:

```text
/root/sg-panel-backups
```
