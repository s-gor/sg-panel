# Пользователи, прямые ссылки и подписки

## Что является пользователем

Пользователь SG-Panel — это отдельная учётная запись VLESS с собственным UUID.

Рекомендуется создавать отдельного пользователя для:

- каждого человека;
- каждого доверенного устройства;
- временного доступа;
- тестовой конфигурации.

Это позволяет отключить одну учётную запись без изменения остальных.

## Создание пользователя

Откройте:

```text
Пользователи
```

Заполните:

```text
Имя
Комментарий, необязательно
Срок действия, необязательно
```

Нажмите **«Создать пользователя»**.

SG-Panel автоматически:

1. создаёт UUID;
2. сохраняет пользователя в SQLite;
3. формирует новый `config.json`;
4. выполняет `xray run -test`;
5. применяет конфигурацию;
6. оставляет вас на странице пользователей.

Ожидаемое сообщение:

```text
Пользователь ИМЯ добавлен. Настройки сохранены и применены. Xray работает.
```

## Прямая VLESS-ссылка

В строке пользователя нажмите:

```text
Ссылка / QR
```

Прямая ссылка начинается с:

```text
vless://
```

Она содержит:

- UUID;
- домен и порт;
- transport;
- security;
- SNI;
- XHTTP Path или параметры REALITY;
- имя пользователя в конце ссылки.

Используйте прямую ссылку для разового импорта одного профиля.

## Персональная подписка

Откройте:

```text
Подписки
```

У пользователя должен быть статус:

```text
Включена
```

Нажмите **«Копировать»** рядом с полем **URL подписки**.

URL подписки начинается с:

```text
https://
```

Типовой вид:

```text
https://ДОМЕН:ПОРТ/sub/СЕКРЕТНЫЙ_ТОКЕН
```

Не публикуйте этот URL. Токен предоставляет доступ к конфигурации пользователя.

## Главное различие

| Что скопировано | Начало | Куда вставлять |
|---|---|---|
| Прямая VLESS-ссылка | `vless://` | импорт одного профиля |
| URL подписки | `https://` | группа подписок клиента |

Если в поле URL группы подписок вставить `vless://`, клиент получит не подписку, а обычную конфигурацию в неправильном месте.

## Проверка в браузере

К URL подписки можно временно добавить:

```text
?format=plain
```

Браузер должен показать строку `vless://...`.

Полный URL и токен не отправляйте в сообщения и не публикуйте в логах.

## v2rayN

Откройте настройки группы подписки и заполните:

```text
Remarks: имя пользователя или группы
URL: персональный HTTPS-URL из страницы Подписки
Automatic update interval: подходящий интервал
```

Остальные поля для обычной подписки SG-Panel можно оставить пустыми.

После сохранения выполните обновление подписок.

Ожидаемый результат:

- появляется профиль пользователя;
- имя соответствует пользователю SG-Panel;
- параметры соответствуют текущему Inbound;
- подключение работает.

Названия пунктов меню v2rayN могут немного отличаться между версиями.

## v2rayNG

Создайте группу подписки и вставьте персональный HTTPS-URL.

После обновления должен появиться текущий профиль пользователя.

Для прямого разового подключения можно вместо подписки отсканировать QR-код на странице пользователя.

## v2rayNG и TUN

Для проверки TUN в v2rayNG:

1. выберите режим VPN;
2. включите Hev TUN;
3. на время диагностики отключите per-app proxy;
4. используйте глобальную маршрутизацию;
5. полностью переподключите профиль после обновления подписки.

Серверный DNS SG-Panel для обычного подключения включать не требуется. DNS режима TUN обычно обрабатывает клиент.

Названия переключателей могут отличаться между версиями v2rayNG.

## Что происходит после смены Inbound

UUID пользователя сохраняется, но transport-параметры меняются.

При использовании прямой ссылки:

1. откройте **Пользователи**;
2. получите новую ссылку или QR-код;
3. замените старый профиль в клиенте.

При использовании подписки:

1. оставьте тот же URL подписки;
2. обновите группу подписок в клиенте;
3. клиент получит новую конфигурацию автоматически.

## Счётчик обращений

На странице **Подписки** колонка **«Обращения»** увеличивается при запросе персонального URL.

Счётчик подтверждает выдачу подписки, но не подтверждает, что пользователь успешно подключился к Xray.

## Отключение подписки

Кнопка **«Отключить»** запрещает выдачу конфигурации по URL подписки.

Она не отключает самого пользователя Xray. Для полного запрета доступа отключите пользователя на странице **Пользователи**.

## Новый токен

Кнопка **«Новый токен»**:

- создаёт новый URL подписки;
- немедленно делает старый URL недействительным;
- не меняет UUID пользователя;
- не меняет прямую VLESS-ссылку.

Используйте её, если URL подписки попал к постороннему.

## Глобальные настройки подписок

На странице **Подписки** можно включить или выключить публичную выдачу подписок целиком.

Изменение этих настроек не требует применения `config.json`: ответ формируется веб-панелью из SQLite при каждом запросе.

## Безопасность

- используйте HTTPS;
- ограничьте порт панели своим IP, если подписки не должны загружаться из других сетей;
- для мобильных клиентов настройте отдельный subscription allowlist или откройте порт панели только при необходимости;
- не размещайте токены подписок в публичных репозиториях;
- после утечки нажмите **«Новый токен»**.
