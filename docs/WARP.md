# Cloudflare WARP в SG-Panel

SG-Panel создаёт WARP как встроенный WireGuard-outbound Xray с тегом:

```text
warp
```

Отдельный системный `warp-cli`, интерфейс `wg-quick@warp` и изменение маршрутов Ubuntu не требуются. Туннель существует внутри Xray и используется только тем трафиком, который выбран в маршрутизации SG-Panel.

## Где что настраивается

- **Outbounds** — создание, проверка, включение, пересоздание и удаление WARP Outbound;
- **Routing** — выбор режима: не использовать WARP, выбранные сайты или весь трафик;
- **Xray Config** — итоговая проверка и применение конфигурации.


## Схема

```text
Клиент
  |
  | основной inbound SG-Panel
  v
Xray
  |
  | outbound: warp
  v
Cloudflare WARP
  |
  v
Интернет
```

WARP не меняет основной inbound. Можно продолжать использовать:

```text
RAW/TCP + REALITY
XHTTP + TLS
XHTTP + REALITY
```

## Создание WARP

Откройте:

```text
Выходы
```

В блоке **Cloudflare WARP** нажмите:

```text
Создать WARP
```

SG-Panel выполнит следующие действия:

1. зарегистрирует учётные данные Cloudflare через `wgcf-cli`;
2. создаст native WireGuard-outbound Xray;
3. сохранит секретные данные в SQLite и в `/etc/xpanel-mvp/warp/wgcf.json`;
4. установит тег выхода `warp`;
5. включит `noKernelTun`, чтобы WARP не менял системные маршруты Ubuntu.

Секретный ключ WARP в интерфейсе не показывается.

После создания откройте страницу **Xray Config**, выполните проверку и примените конфигурацию.

## Режимы маршрутизации

### Не использовать

WARP остаётся настроенным и доступным как outbound, но автоматическое правило выключено.

```text
Режим: Не использовать
```

Этот вариант удобен, если выход `warp` назначается вручную в обычных правилах маршрутизации.

### Только выбранные сайты

```text
Режим: Только выбранные сайты
```

SG-Panel создаёт правило:

```text
Название: Cloudflare WARP
Приоритет: 40
Outbound: warp
Network: tcp,udp
```

В поле доменов можно использовать обычные Xray-условия:

```text
domain:google.com
domain:spotify.com
domain:reddit.com
geosite:instagram
geosite:meta
```

По умолчанию SG-Panel предлагает набор для Google, Spotify, Reddit, Meta и OpenAI. Его можно полностью изменить.

### Весь трафик

```text
Режим: Весь трафик
```

Выход `warp` становится выходом по умолчанию. Отдельное выборочное правило отключается.

Системные правила с более высоким приоритетом, например блокировка BitTorrent, продолжают выполняться раньше маршрута по умолчанию.

## Проверка WARP

После применения конфигурации вернитесь в раздел **Outbounds** и нажмите:

```text
Проверить WARP
```

SG-Panel временно запускает отдельный локальный SOCKS-вход Xray, отправляет запрос к Cloudflare trace через WARP и показывает:

```text
состояние WARP
выходной IP
время проверки
```

Рабочий Xray и его основной inbound для этой проверки не останавливаются.

Ожидаемый результат:

```text
WARP on, IP ...
```

или:

```text
WARP plus, IP ...
```

## Отключение

Кнопка **«Отключить»**:

- исключает `warp` из создаваемого `config.json`;
- возвращает выход по умолчанию на `direct`, если использовался WARP;
- отключает правила, ссылающиеся на `warp`;
- сохраняет учётные данные для последующего включения.

После отключения необходимо снова применить конфигурацию.

## Пересоздание

Кнопка **«Пересоздать»** регистрирует новый WARP-профиль и заменяет ключи и адреса WireGuard.

Используйте её, если:

- профиль перестал подключаться;
- Cloudflare отклонил старые учётные данные;
- требуется полностью новый WARP-профиль.

После пересоздания снова проверьте и примените конфигурацию.

## Удаление

Кнопка **«Удалить»**:

- удаляет outbound `warp`;
- удаляет связанные правила маршрутизации;
- возвращает выход по умолчанию на `direct`;
- удаляет `/etc/xpanel-mvp/warp/wgcf.json`;
- очищает сохранённые WARP-данные из SQLite.

После удаления примените конфигурацию.

## Резервное копирование

Учётные данные WARP находятся в базе SG-Panel и каталоге:

```text
/etc/xpanel-mvp/warp
```

Установщик RC5 сохраняет этот каталог при обновлении. Файл учётной записи создаётся с правами `0600`.

## Возможные ошибки

### Не найден wgcf-cli

Повторно запустите установку или обновление RC5. Установщик помещает компонент сюда:

```text
/usr/local/bin/wgcf-cli
```

### WARP создан, но сайты используют IP сервера

Проверьте три действия:

1. WARP включён;
2. выбран режим маршрутизации;
3. конфигурация проверена и применена.

### Выборочное правило не срабатывает

Проверьте порядок правил на странице **Routing**. Xray использует первое совпавшее правило.

### Проверка показывает warp=off

Пересоздайте WARP и примените конфигурацию. Также убедитесь, что сервер может устанавливать исходящие UDP-соединения.

## Endpoint на EC2

SG-Panel использует проверенный IPv4 endpoint:

```text
162.159.192.1:2408
```

Это исключает ситуацию, когда `engage.cloudflareclient.com` разрешается одновременно в IPv4 и IPv6, а Xray выбирает IPv6 на EC2 без IPv6 default route. В таком случае WARP-тест зависал до тайм-аута, хотя учётные данные и JSON были корректны.
