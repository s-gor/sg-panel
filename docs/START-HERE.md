# С чего начать

Эта страница объясняет логику SG-Panel и показывает, в каком разделе выполнять каждую задачу.

## Главная логика

Настройка состоит из трёх независимых частей:

| Вопрос | Раздел |
|---|---|
| Как клиент подключается к серверу? | **Inbound** |
| Какие выходы доступны серверу? | **Outbounds** |
| Какой трафик направлять в каждый выход? | **Routing** |

Итоговый путь трафика всегда имеет вид:

```text
Клиент — Inbound — Xray — Routing — Outbound — Интернет
```

## Карта панели

### Обзор

Отвечает на вопрос: **всё ли сейчас работает?**

Показывает:

- состояние Xray;
- текущий Inbound;
- Default Outbound;
- активных пользователей;
- трафик;
- время последнего применения.

На странице нет служебных кнопок. Для поиска причины ошибки используйте **Diagnostics**.

### Пользователи

Здесь создаются учётные записи Xray.

Для каждого пользователя доступны:

- отдельный UUID;
- срок действия;
- включение и отключение;
- прямая VLESS-ссылка;
- QR-код.

Изменения пользователей применяются к Xray автоматически.

### Подписки

Здесь создаются персональные HTTPS-URL подписок.

Важно различать:

```text
Прямая ссылка:  vless://...
Подписка:       https://ДОМЕН:ПОРТ/sub/ТОКЕН
```

Подписки формируются из SQLite при каждом запросе. Перезапуск Xray не требуется.

### Inbound

Определяет способ входящего подключения:

- `RAW/TCP + REALITY`;
- `XHTTP + TLS`;
- `XHTTP + REALITY`.

После переключения UUID пользователей сохраняются, но клиентские ссылки меняются. Получите новую ссылку или обновите подписку.

### Outbounds

Содержит выходы:

- `direct` — интернет через IP сервера;
- `blocked` — блокировка;
- `warp` — Cloudflare WARP;
- пользовательские VLESS-выходы на другие серверы.

Создание Outbound ещё не означает, что через него пойдёт трафик. Выбор выполняется в **Routing**.

### Routing

Определяет:

- Default Outbound;
- порядок правил;
- выборочный WARP;
- блокировку рекламы и BitTorrent;
- маршруты по доменам, IP, портам, протоколам и пользователям.

Xray использует первое совпавшее правило. Меньший приоритет выполняется раньше.

### DNS

Нужен для специальных DNS-серверов, hosts и сложных схем с `geoip`.

Для обычного сервера DNS Xray можно оставить выключенным.

### Xray Config

Расширенный раздел.

Здесь можно:

- увидеть итоговый `config.json`;
- выполнить `xray run -test`;
- изменить полный JSON;
- синхронизировать JSON обратно с формами.

Обычная работа через формы не требует заходить на эту страницу.

### Diagnostics

Используется только при проверке и поиске ошибок.

Содержит:

- состояние Xray, Nginx и SG-Panel;
- проверку WARP;
- текущие порты;
- журналы служб;
- системный DNS;
- диагностический отчёт.

### Резервные копии

Создаёт копии:

- `panel.db`;
- текущего `config.json`.

В базе находятся UUID, Reality private key, WARP и токены подписок. Храните копии как секретные данные.

### Безопасность

Содержит:

- пароль администратора;
- активные сессии;
- IP allowlist;
- ограничения для подписок;
- журнал действий.

## Обычные сценарии

### Добавить пользователя

```text
Пользователи — Создать пользователя — Ссылка / QR
```

Применение выполняется автоматически.

### Выдать подписку

```text
Подписки — пользователь — Копировать URL подписки
```

В клиент подписок вставляется HTTPS-URL, а не `vless://`.

### Переключить основной протокол

```text
Inbound — выбрать профиль — проверить поля — Сохранить и применить
```

После переключения обновите ссылку или подписку в клиенте.

### Включить весь трафик через WARP

```text
Outbounds — Создать WARP — Проверить WARP
Routing — Весь трафик через WARP — Сохранить и применить
```

### Направить один сайт через WARP

```text
Routing — Только выбранные сайты — указать домены — Сохранить и применить
```

### Вернуть выход через IP сервера

```text
Routing — Не использовать WARP — Сохранить и применить
```

Default Outbound станет `direct`.

## Когда применяется конфигурация

Автоматически применяются:

- пользователи;
- Inbound;
- Outbounds;
- WARP;
- Routing;
- DNS.

Не требуют применения Xray:

- подписки;
- безопасность;
- создание и скачивание резервной копии;
- тест WARP;
- просмотр Diagnostics.

Намеренно работают в два шага:

- полный JSON на странице **Xray Config**;
- восстановление базы из резервной копии.
