# Routing и гео-правила

Routing определяет, какой трафик направляется в каждый Outbound.

## Default Outbound

Default Outbound используется, если ни одно явное правило не совпало.

Примеры:

```text
direct    интернет через IP сервера
warp      интернет через Cloudflare WARP
eu-exit   интернет через пользовательский сервер
```

Создание Outbound не меняет Default Outbound автоматически.

## Порядок правил

Xray проверяет правила по возрастанию приоритета и использует первое совпавшее.

```text
10  Блокировать BitTorrent
20  Блокировать рекламу
30  Локальные сети напрямую
40  Франция через eu-exit
```

После создания, изменения, включения или удаления правила SG-Panel сразу проверяет и применяет конфигурацию.

## Условия внутри одного правила

Значения внутри одного поля работают как варианты.

Разные поля должны совпасть одновременно.

Пример:

```text
Domain:   domain:example.com
Port:     443
Network:  tcp
```

Правило применяется только к TCP-соединениям на порт `443` для указанного домена.

Поэтому страна создаётся двумя правилами:

```text
geosite:fr
geoip:fr
```

SG-Panel делает это автоматически в шаблоне страны.

## Форматы доменов

```text
domain:example.com
full:api.example.com
keyword:example
geosite:category-ads-all
geosite:fr
```

## Форматы IP

```text
1.1.1.1
203.0.113.0/24
geoip:private
geoip:fr
```

## Готовые правила

На странице **Routing** доступны шаблоны:

- блокировка рекламы;
- локальные сети;
- страна: домены и IP;
- произвольная категория `geosite`.

Для страны используется двухбуквенный код:

```text
fr
de
us
ru
```

## WARP

Режимы WARP находятся на странице **Routing**:

```text
Не использовать WARP
Только выбранные сайты
Весь трафик через WARP
```

При режиме всего трафика `warp` становится Default Outbound. Отдельная строка WARP в таблице правил не обязательна.

## Sniffing

Sniffing позволяет Xray распознавать домены и протоколы внутри соединений.

Для обычной схемы рекомендуется:

```text
Sniffing:    включён
Route only:  включён
HTTP:        включён
TLS:         включён
QUIC:        включён
```

`Route only` использует распознанный домен для Routing, но не заменяет адрес назначения в запросе.

## Domain strategy

- `AsIs` — использовать домен как есть;
- `IPIfNonMatch` — разрешить домен в IP, если доменные правила не совпали;
- `IPOnDemand` — разрешать домены при необходимости IP-проверки.

Для простых доменных правил используйте `AsIs`.

Если правила `geoip` должны срабатывать для доменных запросов, может потребоваться `IPIfNonMatch` или `IPOnDemand`.

## Геоданные

SG-Panel показывает наличие:

```text
geoip.dat
geosite.dat
```

Файлы ищутся в:

```text
$XRAY_LOCATION_ASSET
/usr/local/share/xray
/usr/share/xray
```

Если нужного файла нет, `xray run -test` может завершиться ошибкой.

## Outbound и balancer

Обычная цель:

```json
"outboundTag": "eu-exit"
```

Продвинутая цель:

```json
"balancerTag": "europe"
```

Балансировщики редактируются через полный JSON Routing.

Пример:

```json
{
  "domainStrategy": "AsIs",
  "balancers": [
    {
      "tag": "europe",
      "selector": ["eu-", "de-", "fr-"]
    }
  ],
  "rules": [
    {
      "_sgPanel": {
        "name": "Европейские сайты",
        "priority": 50,
        "enabled": true
      },
      "type": "field",
      "domain": ["geosite:fr", "geosite:de"],
      "balancerTag": "europe"
    }
  ]
}
```

## Формы и JSON

Доступны:

1. форма отдельного правила;
2. JSON отдельного правила;
3. полный JSON объекта Routing.

Каждое сохранение правила или полного Routing JSON автоматически применяется к Xray.

Дополнительные поля, добавленные через JSON, сохраняются при последующем изменении известных полей через форму.

## Проверка результата

Откройте **Обзор** и проверьте Default Outbound.

Для подробной проверки откройте **Diagnostics**:

- `config.json` должен иметь состояние `OK`;
- Xray должен быть `active`;
- журналы не должны содержать новых ошибок Routing.
