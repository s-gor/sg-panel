# Маршрутизация и гео-правила

## Порядок правил

Xray проверяет routing rules сверху вниз и использует первое совпавшее правило. В SG-Panel порядок определяется полем **«Приоритет»**: меньшее число выполняется раньше.

Пример:

```text
10  Блокировать BitTorrent
20  Блокировать рекламу
30  Локальные сети напрямую
40  Франция через eu-exit
```

## Условия внутри одного правила

Разные поля одного rule не означают «или». Если одновременно указать `domain` и `ip`, Xray рассматривает их как совместные условия.

Поэтому правило для страны создаётся двумя отдельными правилами:

```json
{
  "type": "field",
  "domain": ["geosite:fr"],
  "outboundTag": "eu-exit"
}
```

```json
{
  "type": "field",
  "ip": ["geoip:fr"],
  "outboundTag": "eu-exit"
}
```

SG-Panel делает это автоматически в шаблоне **«Страна: домены + IP»**.

## Готовые гео-правила

На странице **Routing** доступны шаблоны:

- блокировать рекламу — `geosite:category-ads-all`;
- локальные сети — `geoip:private`;
- страна — `geosite:cc` и `geoip:cc`;
- произвольная категория geosite.

Для страны вводится двухбуквенный код, например:

```text
fr
de
us
ru
```

## Геоданные

На странице отображается состояние:

```text
geoip.dat
geosite.dat
```

SG-Panel ищет файлы в:

```text
$XRAY_LOCATION_ASSET
/usr/local/share/xray
/usr/share/xray
```

Если файл не найден, соответствующие правила могут завершить проверку Xray ошибкой.

## Domain strategy

- `AsIs` — использовать доменное имя как есть;
- `IPIfNonMatch` — разрешить домен в IP, если доменные правила не совпали;
- `IPOnDemand` — разрешать домены при необходимости IP-проверки.

Если используются правила `geoip`, а трафик приходит как доменное имя, обычно удобнее `IPIfNonMatch` или `IPOnDemand`.

## Outbound и balancer

Обычное правило направляет трафик через:

```json
"outboundTag": "eu-exit"
```

Продвинутая конфигурация может использовать:

```json
"balancerTag": "europe"
```

Балансировщики добавляются через полный JSON routing:

```json
{
  "domainStrategy": "AsIs",
  "balancers": [
    {
      "tag": "europe",
      "selector": ["eu-", "de-", "fr-"]
    }
  ],
  "rules": [
    {
      "_sgPanel": {
        "name": "Европейские сайты",
        "priority": 50,
        "enabled": true
      },
      "type": "field",
      "domain": ["geosite:fr", "geosite:de"],
      "balancerTag": "europe"
    }
  ]
}
```

## Формы и JSON

Доступны три уровня:

1. форма отдельного правила;
2. JSON отдельного правила;
3. полный JSON объекта routing.

Все три варианта работают с одной базой. Дополнительные поля, добавленные в JSON, сохраняются при изменении известных полей через форму.

## Проверка

После изменения:

1. откройте **Xray Config**;
2. проверьте итоговый JSON;
3. нажмите **«Проверить»**;
4. убедитесь, что Xray сообщает `Configuration OK`;
5. нажмите **«Применить конфигурацию»**.
