# Руководство по использованию SG-Panel

Это руководство описывает базовую работу с пользователями, маршрутизацией, фильтрацией, пользовательскими выходами и DNS.

## Главная страница

После входа убедитесь, что:

- SG-Panel имеет состояние `active`;
- Nginx имеет состояние `active`;
- Xray имеет состояние `active`;
- конфигурация Xray проходит проверку.

После серьёзных изменений возвращайтесь на главную страницу и проверяйте состояние Xray.

## Пользователи

Для каждого человека или устройства рекомендуется создавать отдельного пользователя. Это позволяет независимо:

- отключать доступ;
- задавать срок действия;
- получать отдельную VLESS-ссылку;
- показывать отдельный QR-код;
- удалять потерянные или больше не используемые подключения.

Первый пользователь `sg-admin` является обычным пользователем Xray и не является администратором панели.

После добавления, изменения, включения или отключения пользователя нажмите **«Применить»**.

## Рекомендуемые настройки маршрутизации

Для обычного сервера без пользовательских выходов рекомендуется оставить:

```text
Default outbound: direct
Domain strategy:  AsIs
Sniffing:         включён
Route only:       включён
HTTP:             включён
TLS:              включён
QUIC:             включён
```

`direct` направляет трафик непосредственно через текущий сервер.

`blocked` используется для блокировки выбранного трафика.

После изменения параметров нажмите **«Сохранить настройки»**, а затем **«Применить»**.

## Как работают правила

Правила выполняются по приоритету:

```text
10 выполняется раньше 20
20 выполняется раньше 100
```

Xray использует первое совпавшее правило. Более точные правила должны иметь меньший номер, а общие — больший.

Несколько значений внутри одного поля работают как варианты:

```text
domain:example.com
domain:example.org
```

Правило сработает для любого из этих доменов.

Если заполнено несколько разных полей, они должны совпасть одновременно.

Пример:

```text
Domains: domain:example.com
Ports:   443
Network: tcp
```

Такое правило применяется только к TCP-соединениям на порт `443` для указанного домена.

Пустые поля в проверке не участвуют.

## Фильтрация по доменам

Основные форматы:

```text
domain:example.com
```

Домен и его поддомены.

```text
full:api.example.com
```

Точное полное имя.

```text
keyword:example
```

Совпадение по части имени.

```text
geosite:category-ads-all
```

Готовая категория geosite.

Каждое значение удобнее указывать с новой строки.

## Фильтрация по IP

Один IP:

```text
1.1.1.1
```

Диапазон CIDR:

```text
203.0.113.0/24
```

Категория GeoIP:

```text
geoip:private
```

`geoip:private` запрещает доступ к частным и локальным сетям. Не используйте его, если через Xray нужен доступ к внутренней сети.

## Фильтрация по портам и протоколам

Один порт:

```text
443
```

Несколько портов:

```text
80,443,853
```

Диапазон:

```text
1000-2000
```

Сети:

```text
tcp
udp
tcp,udp
```

Протоколы:

```text
http
tls
quic
bittorrent
```

## Пример блокировки BitTorrent

```text
Название:   Block BitTorrent
Приоритет:  10
Outbound:   blocked
Protocols:  bittorrent
```

Остальные поля оставьте пустыми.

Сохраните правило и нажмите **«Применить»**.

## Пример блокировки рекламы

```text
Название:   Block ads
Приоритет:  20
Outbound:   blocked
Domains:    geosite:category-ads-all
```

Такая блокировка не заменяет полноценный фильтр браузера и может нарушать работу отдельных сайтов.

## Правило для конкретного пользователя

В поле **«Пользователи»** можно выбрать одного или нескольких пользователей.

Например, правило блокировки BitTorrent можно применить только к одному пользователю.

Для множественного выбора используйте:

```text
Ctrl       Windows и Linux
Command    macOS
```

## Пользовательские выходы

Встроенные системные выходы `direct` и `blocked` доступны всегда. Дополнительно можно создать несколько пользовательских VLESS-выходов, каждый со своим тегом и собственными параметрами удалённого сервера.

Поддерживаемые комбинации:

```text
VLESS + RAW/TCP + REALITY
VLESS + XHTTP + TLS
VLESS + XHTTP + REALITY
```

После создания выход можно назначить выходом по умолчанию или выбрать только в отдельных правилах маршрутизации.

Пошаговое заполнение всех полей, перенос параметров из VLESS-ссылки и пример подключения к серверу из проекта `xray-xhttp-v2` описаны отдельно:

[Создание пользовательских выходов](OUTBOUNDS.md)

Кнопка проверки TCP подтверждает только доступность адреса и порта. Она не проверяет UUID, XHTTP Path, TLS, REALITY и остальные параметры соединения.

## DNS

В стандартной установке внутренний DNS SG-Panel можно оставить выключенным.

Расширенные DNS-настройки нужны только для специальных схем маршрутизации, отдельных DNS-серверов или устранения DNS-утечек.

Без необходимости их менять не следует.

## Проверка перед применением

После серьёзных изменений:

1. откройте предварительный просмотр;
2. проверьте сформированный JSON;
3. нажмите **«Применить»**;
4. убедитесь на главной странице, что Xray имеет состояние `active`.

Перед крупными изменениями создайте резервную копию.
