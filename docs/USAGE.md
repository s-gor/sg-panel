# Ежедневное использование SG-Panel

Эта страница описывает обычную работу после завершённой установки.

## Перед началом

Откройте **Обзор**.

Нормальное состояние:

```text
Xray              active
Inbound           выбранный профиль
Default Outbound  direct или warp
Пользователи      активных / всего
```

Если страница показывает **«Требуется внимание»**, сначала откройте **Diagnostics**.

## Как применяется обычное изменение

Для пользователей, Inbound, Outbounds, Routing и DNS используется один цикл:

1. изменение сохраняется в SQLite;
2. SG-Panel собирает итоговый `config.json`;
3. Xray проверяет конфигурацию;
4. создаётся резервная копия;
5. новый конфиг применяется;
6. служба проверяется;
7. при ошибке возвращается предыдущий рабочий конфиг.

Не нужно переходить на **Обзор** или **Xray Config** и нажимать вторую кнопку.

## Пользователи

Для каждого человека или устройства создавайте отдельного пользователя.

Это позволяет независимо:

- отключать доступ;
- задавать срок действия;
- менять UUID;
- выдавать прямую ссылку;
- выдавать персональную подписку.

Кнопки создания, изменения, включения, отключения и удаления сразу применяют конфигурацию Xray.

Подробно: [Пользователи, ссылки и подписки](USERS-AND-SUBSCRIPTIONS.md).

## Переключение Inbound

Откройте **Inbound**, выберите профиль и нажмите **«Сохранить и применить»**.

После успешного переключения:

- UUID пользователей сохраняются;
- Routing и Outbounds сохраняются;
- старая прямая ссылка может перестать работать;
- подписка остаётся прежней, но её нужно обновить в клиенте.

Подробно: [Основные Inbound-профили](INBOUND-PROFILES.md).

## Default Outbound

Default Outbound используется, когда ни одно правило Routing не совпало.

Основные варианты:

```text
direct   интернет через IP сервера
warp     интернет через Cloudflare WARP
```

Пользовательский Outbound также можно назначить выходом по умолчанию.

Выбор выполняется на странице **Routing**, а не **Outbounds**.

## Порядок правил Routing

Xray использует первое совпавшее правило.

Меньшее значение приоритета выполняется раньше:

```text
10  блокировка BitTorrent
20  блокировка рекламы
40  выборочный WARP
100 общее правило
```

Если заполнено несколько разных условий одного правила, они должны совпасть одновременно.

Пример:

```text
Domain:   domain:example.com
Port:     443
Network:  tcp
```

Правило сработает только для TCP-подключения к `example.com:443`.

## Доменные условия

```text
domain:example.com
full:api.example.com
keyword:example
geosite:category-ads-all
```

Каждое значение указывайте с новой строки.

## IP-условия

```text
1.1.1.1
203.0.113.0/24
geoip:private
geoip:fr
```

Не блокируйте `geoip:private`, если через Xray нужен доступ к локальным сетям.

## Порты, сети и протоколы

Порты:

```text
443
80,443,853
1000-2000
```

Сети:

```text
tcp
udp
tcp,udp
```

Распознаваемые протоколы зависят от sniffing. Часто используются:

```text
http
tls
quic
bittorrent
```

## Блокировка BitTorrent

Создайте правило:

```text
Название:   Блокировка BitTorrent
Приоритет:  10
Выход:      blocked
Protocol:   bittorrent
```

После нажатия основной кнопки правило применяется автоматически.

## Блокировка рекламы

Создайте правило:

```text
Название:   Блокировка рекламы
Приоритет:  20
Выход:      blocked
Domain:     geosite:category-ads-all
```

Такая блокировка не заменяет расширение браузера и может нарушать отдельные сайты.

## Правило для одного пользователя

В поле пользователей выберите нужные учётные записи.

Для множественного выбора:

```text
Ctrl       Windows и Linux
Command    macOS
```

Правило будет действовать только на выбранных пользователей.

## WARP

Правильный порядок:

1. **Outbounds** — создать WARP;
2. **Outbounds** или **Diagnostics** — проверить WARP;
3. **Routing** — выбрать режим;
4. **Обзор** — убедиться, что Default Outbound изменился.

Не включайте весь трафик через WARP, пока отдельная проверка не показывает `WARP on` или `WARP plus`.

Подробно: [Cloudflare WARP](WARP.md).

## Пользовательские Outbounds

Создание Outbound только добавляет доступный выход. Сам трафик не переключается автоматически.

После создания:

- назначьте его Default Outbound на странице **Routing**;
- или создайте отдельное правило Routing.

Подробно: [Пользовательские Outbounds](OUTBOUNDS.md).

## DNS

Для обычного сервера оставьте DNS Xray выключенным.

Изменяйте DNS, если требуется:

- отдельный DNS-сервер для выбранных доменов;
- специальные hosts;
- контроль fallback;
- схема с `geoip` и принудительным разрешением домена.

Изменения DNS применяются автоматически.

## Xray Config

Обычная работа не требует ручного редактирования JSON.

Используйте **Xray Config**, когда нужно:

- увидеть итоговый документ;
- проверить нестандартное поле;
- добавить параметр Xray, которого нет в форме;
- синхронизировать полный JSON обратно с панелью.

Полный JSON намеренно сохраняется и применяется в два этапа. Это защита от случайной замены всей конфигурации.

## Резервные копии

Создавайте ручную копию перед:

- большим изменением Routing;
- редактированием полного JSON;
- сменой нескольких Outbounds;
- экспериментами с DNS.

После восстановления базы проверьте Inbound, Outbounds и Routing, затем примените конфигурацию через соответствующую страницу или **Xray Config**.
