# Changelog

## 0.10.0 RC8 — Clean install namespace fix

- исправлена чистая установка: `deploy/install-service.sh` заранее создаёт `/var/www/sg-panel-placeholder` и остальные каталоги из `ReadWritePaths` до запуска `xpanel-web`;
- устранена ошибка systemd `status=226/NAMESPACE` после полной очистки или на новом EC2;
- добавлен регрессионный тест, проверяющий порядок создания каталогов до записи unit-файла;
- функциональность RC7 сохранена без изменений.

## 0.10.0 RC7 — One-click workflow and clear diagnostics

- Dashboard превращён в чистый обзор: состояние Xray, текущий Inbound, Default Outbound, пользователи, трафик и время последнего применения;
- повторяющиеся кнопки применения, проверки JSON и перезапуска удалены из Dashboard;
- Diagnostics теперь отвечает за службы, WARP, сеть, технические параметры, перезапуск Xray/Nginx/SG-Panel и журналы;
- на страницах Inbound, Outbounds, Routing, Users и DNS изменения сразу проходят сборку, `xray run -test`, резервное копирование и применение;
- основные кнопки переименованы в «Сохранить и применить»; переход на Dashboard и второе нажатие больше не требуются;
- сообщения об успехе и ошибках показываются поверх текущей страницы и остаются видимыми независимо от положения прокрутки;
- WARP endpoint принудительно закреплён на проверенном IPv4 `162.159.192.1:2408`, чтобы EC2 без IPv6 default route не выбирал недоступный IPv6;
- создание и пересоздание WARP теперь также сохраняет рабочий IPv4 endpoint;
- добавлены регрессионные проверки нового workflow, разделения Dashboard/Diagnostics и IPv4 endpoint WARP;
- полный набор: 139 автоматических тестов.

## 0.10.0 RC6 — WARP default outbound fix

- исправлен порядок `outbounds` при наличии сохранённого базового JSON: выбранный default outbound теперь действительно становится первым в итоговом `config.json`;
- добавлен регрессионный тест для режима «Весь трафик через WARP» после обновления существующей конфигурации;
- на странице Routing явно показывается текущий default outbound и объясняется отсутствие отдельного правила в режиме всего трафика.

## 0.10.0 RC5 — Clear Inbound / Outbounds / Routing workflow

- технические разделы интерфейса получили однозначные названия: `Inbound`, `Outbounds`, `Routing` и `Xray Config`;
- навигация выстроена по рабочей последовательности: Inbound → Outbounds → Routing → DNS → Xray Config;
- управление WARP Outbound оставлено на странице Outbounds, а режимы маршрутизации WARP перенесены на страницу Routing;
- смена пароля перенесена со страницы Inbound в раздел безопасности;
- глобальные параметры Xray, журналирование, Stats API и системные пути перенесены в Xray Config;
- страница Inbound теперь отвечает только за входящее подключение, транспорт, REALITY/TLS и параметры клиентских ссылок;
- три карточки inbound располагаются в одну строку на широком экране; на среднем третья карточка занимает всю ширину, на телефоне используется одна колонка;
- исправлен шаблон Routing: JavaScript больше не попадает внутрь HTML-заголовка страницы;
- добавлены проверки структуры навигации, разделения WARP и сохранения служебных параметров.


## 0.10.0 RC4 — Verified inbound profiles and Cloudflare WARP

- на реальном EC2 подтверждено переключение без JSON между `RAW/TCP + REALITY`, `XHTTP + TLS` и `XHTTP + REALITY` с сохранением пользователя и UUID;
- `gRPC + TLS` удалён из основных карточек, поскольку Xray 26.3.27 сообщает об устаревании gRPC transport и рекомендует XHTTP stream-up H2; backend и полный JSON сохраняют совместимость с уже существующей gRPC-конфигурацией;
- добавлен встроенный Cloudflare WARP: регистрация через `wgcf-cli`, native WireGuard-outbound Xray с тегом `warp`, включение, отключение, пересоздание, удаление и проверка выходного IP;
- добавлены режимы WARP routing: выключен, только выбранные домены и весь трафик;
- WARP использует `noKernelTun`, не создаёт системный WireGuard-интерфейс и не изменяет маршруты Ubuntu;
- установщик загружает `wgcf-cli v0.3.6`, проверяет SHA-256 и сохраняет WARP-учётные данные при обновлении;
- исправлена systemd-защита `xpanel-web`: в `ReadWritePaths` добавлены `/etc/nginx` и `/var/www/sg-panel-placeholder`;
- при переключении на `XHTTP + TLS` SNI автоматически меняется на публичный домен, а пути сертификата и private key подставляются из `/etc/letsencrypt/live/ДОМЕН/`;
- при возврате к REALITY SNI автоматически берётся из Reality target, а Flow включается только для `RAW/TCP + REALITY`;
- ошибки записи и применения теперь обрабатывают `OSError` и показываются в интерфейсе вместо необработанного `Internal Server Error`;
- значение Reality target/SNI по умолчанию остаётся `www.bing.com`, проверенное с Xray 26.3.27;
- добавлены инструкции `docs/INBOUND-PROFILES.md` и `docs/WARP.md`, а также автоматические тесты WARP, профилей, ссылок, JSON и Nginx.

## 0.10.0 RC2 — Full config JSON

- страница «Конфиг» получила отдельный полноценный редактор всего `config.json`;
- формы и полный JSON используют одну модель данных: известные секции синхронизируются обратно с GUI;
- синхронизируются основной Reality inbound, пользователи, выходы, routing, DNS, log и статистика;
- дополнительные верхнеуровневые секции и неизвестные параметры Xray сохраняются, включая `observatory`, дополнительные inbounds/outbounds, `sockopt` и другие поля;
- выходы с протоколами, которых ещё нет в формах, отображаются как JSON-выходы и доступны для выбора в маршрутизации;
- перед сохранением полный JSON проверяется через `xray run -test`; при любой ошибке SQLite автоматически восстанавливается;
- отключённые пользователи и выходы не удаляются при сохранении полного JSON; отсутствующие активные объекты безопасно отключаются;
- итоговый предпросмотр config.json остаётся отдельным этапом, а применение к Xray выполняется только кнопкой «Применить конфигурацию»;
- добавлены тесты двусторонней синхронизации полного JSON; полный набор: 93 теста.

## 0.9.8 hotfix — Interrupted install recovery

- повторный запуск после ошибки Certbot снова открывает мастер, а не ошибочно переходит в режим обычного обновления;
- завершённость установки определяется по рабочему HTTPS-конфигу и сертификату, а после успеха записывается отдельный маркер;
- добавлен параметр `--reconfigure` для смены домена, HTTPS-порта, Reality target и Reality SNI без удаления базы, ключей и пользователей;
- загрузчик `install-from-github.sh` передаёт параметры основному мастеру;
- при возобновлении установки существующий пароль администратора сохраняется, а адрес Xray и Reality-параметры обновляются из повторно введённых значений;
- карточка памяти показывает использованный процент и объём по формуле `MemTotal − MemAvailable`;
- полная очистка использует подтверждение `DELETE ALL`, выполняет `cd /` и ожидает освобождения `dpkg`.

## 0.9.8 — Safe uninstall

- обычный `deploy/uninstall.sh` теперь удаляет только SG-Panel и сохраняет Xray, его конфигурацию и резервные копии;
- перед удалением автоматически создаётся финальная резервная копия SQLite, `config.json`, `web.env` и ссылки первого пользователя;
- удаление Xray и резервных копий возможно только через явные параметры `--remove-xray` и `--remove-backups`;
- старый опасный параметр `--purge-all` больше не выполняет удаление;
- полная очистка одноразового тестового сервера вынесена в отдельный `deploy/purge-test-server.sh` и требует явного подтверждения;
- добавлена инструкция `docs/UNINSTALL.md` с точным описанием сохраняемых и удаляемых компонентов;
- README обновлён и больше не предлагает разрушительную команду удаления по умолчанию.

## 0.9.3 — EC2 Ready

- добавлен мастер первой установки `deploy/ec2-first-install.sh`;
- мастер устанавливает Xray 26.3.27, SG-Panel, Nginx и Certbot;
- добавлена проверка A-записей Dynu перед выпуском сертификата;
- HTTPS-порт панели по умолчанию изменён на `61443`;

- backend панели на EC2 слушает только `127.0.0.1:8080`;
- порт `8080` не требуется открывать в AWS Security Group;
- Nginx слушает порт `80` для ACME HTTP-01 и выбранный HTTPS-порт панели;
- добавлен deploy hook для перезагрузки Nginx после обновления сертификата;
- `configure-https.sh` теперь использует порт `61443` по умолчанию;
- `install-gui.sh` поддерживает переменные окружения для bind address и порта;
- сохранены все функции и дизайн версии 0.9.2.
